怎样将python程序及其依赖打包成rpm包部署

發表於 2016-11-01 | | 閱讀次數

背景

之前接触一个项目，采用tornado框架，开发一个数据库中间件，功能是一个容器控制器，容器内业务管理，监控等功能，做为守护进程部署在几千个公司的业务容器上。

公司目前要求所有程序部署必须打包成rpm包安装，公司有自己的rpm私有源。但这对python程序就比较坑，所有pip依赖包，都需要人肉打包成rpm上传到公司私有源。部署时根据rpm依赖自动拉取依赖包。并且安装在全局！！没有采用虚拟环境，禁用pip!!这对python开发来说简直反人类啊！

解决办法

首先把所有项目依赖下载下来，项目本身也通过setup.py打包成tar包。

在本地，新建一个build/usr文件夹，作为pip的安装目标路径，然后使用pip安装（需要指定源地址为本地路径）你的项目，将会把你的项目包及其依赖一股脑到安装到本地build/usr文件夹。

安装完毕，你会看到类似build/usr/lib/python2.7/site-pakges/…这样的结构。

然后开始写rpm打包脚本，指定build为根目录，最后使用rpmbuild工具打包，也可以使用jekins.

将最终的rpm包通过yum install 安装到目标机器，逛逛逛！项目跑起来了！不用直接再一一安装依赖。

这样的好处是，本地开发不用担心依赖包有没有现成的rpm源码包，愉快的使用pip即可。部署的时候，把以上流程写成一个自动化脚本，还是比较爽快的。

注意的地方：
写rpm打包脚本的时候，有一个after_remove.sh脚本，需要写卸载文件，此时，如果将你的项目源码卸载没关系，但你的项目依赖可别完全一股脑卸载了，有可能影响目标机器上的其他项目…..
